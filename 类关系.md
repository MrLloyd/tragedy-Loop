## 二、各类职责速查

| 类 | 文件 | 职责 |
|---|---|---|
| `GameController` | `game_controller.py` | 调度中枢，持有全部组件，驱动主循环 |
| `StateMachine` | `state_machine.py` | 纯流转：15 阶段 + 条件分支，不接触业务数据 |

| `EventBus` | `event_bus.py` | 发布-订阅总线，解耦结算链与日志/能力 |

| `UICallback` | `game_controller.py` | UI 层向引擎注册的回调接口 |

## 三、AtomicResolver 内部结构

```
AtomicResolver
  ├─ 构造依赖
  │    ├─ event_bus: EventBus   (写操作后发布事件)
  │    └─ death_resolver: DeathResolver (处理 character_death mutation)
  │
  ├─ 内部数据类
  │    ├─ Mutation          单个状态变更记录
  │    │    ├─ mutation_type: str   "token_change" | "character_death" |
  │    │    │                       "character_move" | "protagonist_death" |
  │    │    │                       "protagonist_failure" | "force_loop_end" |
  │    │    │                       "reveal_identity" | "ex_gauge_change"
  │    │    ├─ target_id: str       角色 id 或 area_id
  │    │    └─ details: dict        附带数据（delta / destination / cause …）
  │    │
  │    ├─ Trigger           链式触发载体
  │    │    ├─ trigger_type: str    "on_death" | "protagonist_death" |
  │    │    │                       "protagonist_failure" | …
  │    │    ├─ source_id: str       触发来源
  │    │    ├─ is_terminal: bool    是否为终局效果
  │    │    └─ effects: list        待执行的后续 Effect
  │    │
  │    └─ ResolutionResult  结算输出
  │         ├─ mutations: list[Mutation]
  │         ├─ outcome: Outcome     NONE / PROTAGONIST_DEATH /
  │         │                       PROTAGONIST_FAILURE / MASTERMIND_WIN …
  │         └─ announcements: list[str]
  │
  └─ 三步法实现
       ├─ _plan_effect(snapshot, effect) → list[Mutation]   ① 读（不写状态）
       ├─ _apply_mutation(state, mutation)                   ② 写（含调 DeathResolver）
       └─ _process_triggers(state, mutations) → Result      ③ 触发（队列式）
            └─ _adjudicate(terminal_effects, state) → Outcome  同时裁定
```

### 三步法时序

```
resolve(state, effects, sequential=False)
  │
  ├─ [同时生效] _resolve_simultaneous
  │    ├─ ① snapshot = state.snapshot()
  │    │   for effect in effects:
  │    │       planned_mutations += _plan_effect(snapshot, effect)   # 只读快照
  │    │
  │    ├─ ② for mutation in planned_mutations:
  │    │       _apply_mutation(state, mutation)                       # 写真实状态
  │    │       └─ 其中 character_death → DeathResolver.process_death()
  │    │          其余写操作 → event_bus.emit(TOKEN_CHANGED / …)
  │    │
  │    └─ ③ _process_triggers(state, planned_mutations)
  │         ├─ 收集初始 Trigger（death/failure/force_end）
  │         ├─ while trigger_queue:
  │         │    trigger = queue.popleft()
  │         │    if terminal → 放入 terminal_effects 列表
  │         │    else → 递归执行 effects，产生新 mutations/triggers 入队
  │         └─ _adjudicate(terminal_effects) → Outcome
  │
  └─ [顺序生效 sequential=True] _resolve_sequential
       for effect in effects:           # 每个 effect 独立走完三步法
           sub_result = _resolve_simultaneous(state, [effect])
```

### 同时裁定规则（_adjudicate）

```
terminal_effects 中：
  has_death   = 含 protagonist_death trigger
  has_failure = 含 protagonist_failure trigger

  has_death  且 !soldier_protection → Outcome.PROTAGONIST_DEATH
  has_death  且  soldier_protection → protagonist_dead=False
                                      has_failure → PROTAGONIST_FAILURE
                                      else        → NONE
  !has_death 且  has_failure        → PROTAGONIST_FAILURE
  其他                               → NONE
```

## 四、EventBus 内部结构

```
EventBus
  ├─ _handlers: dict[GameEventType, list[EventHandler]]   订阅表
  └─ _log:      list[GameEvent]                            完整事件日志
  │
  ├─ subscribe(event_type, handler)    注册订阅
  ├─ unsubscribe(event_type, handler)  注销订阅
  ├─ emit(event)                       同步广播（先记日志，再逐个回调）
  ├─ clear_handlers()                  清空订阅（换局用）
  └─ log / clear_log()                 裁定日志读写

GameEvent（事件载体）
  ├─ event_type: GameEventType
  └─ data: dict[str, Any]
```

### GameEventType 分类

| 分类 | 事件类型 | 典型 data keys |
|---|---|---|
| 角色 | `CHARACTER_DEATH` | `character_id`, `cause` |
| 角色 | `CHARACTER_REVIVED` | `character_id` |
| 角色 | `CHARACTER_MOVED` | `character_id`, `destination` |
| 角色 | `CHARACTER_REMOVED` | `character_id` |
| 指示物 | `TOKEN_CHANGED` | `target_id`, `token_type`, `delta` |
| 终局 | `PROTAGONIST_DEATH` | `cause` |
| 终局 | `PROTAGONIST_FAILURE` | `cause` |
| 终局 | `LOOP_END_FORCED` | — |
| 信息 | `IDENTITY_REVEALED` | `character_id`, `identity_id` |
| 信息 | `INCIDENT_OCCURRED` | `incident_id`, `day` |
| 阶段 | `PHASE_CHANGED` | — |
| 阶段 | `LOOP_STARTED / LOOP_ENDED / GAME_ENDED` | — |
| 能力 | `ABILITY_DECLARED / ABILITY_REFUSED` | `character_id`, `ability_id` |
| EX | `EX_GAUGE_CHANGED` | `delta`, `new_value` |
| EX | `WORLD_MOVED` (AHR预留) | — |

### EventBus 与其他组件的关系

```
GameController
  └─ 创建 EventBus 实例，传给：
       ├─ AtomicResolver   — 写操作后 emit 指示物/移动/身份等事件
       └─ PhaseHandler     — 阶段内 emit LOOP_ENDED / INCIDENT_OCCURRED 等

AtomicResolver._apply_mutation
  ├─ token_change    → emit TOKEN_CHANGED
  ├─ character_move  → emit CHARACTER_MOVED
  ├─ reveal_identity → emit IDENTITY_REVEALED
  └─ ex_gauge_change → emit EX_GAUGE_CHANGED

AtomicResolver._collect_triggers_from_mutation
  └─ character_death (DIED) → emit CHARACTER_DEATH
     └─ 若存在 ON_DEATH 身份能力 → emit ABILITY_DECLARED

GameController 关键发布点（已接线）：
  ├─ _run_phase()      → emit PHASE_CHANGED
  ├─ _handle_signal()  → ForceLoopEnd 时 emit LOOP_END_FORCED
  ├─ NEXT_LOOP 入口    → emit LOOP_STARTED
  └─ GAME_END 判定后   → emit GAME_ENDED
```

## 五、DeathResolver 责任链

```
DeathResolver.process_death(character, state) → DeathResult

  ① 不死特性检查
  │    character.base_traits 含 Trait.IMMORTAL
  │    → return PREVENTED_BY_IMMORTAL（角色不死）
  │
  ② 护卫指示物检查
  │    character.tokens.guard >= 1
  │    → tokens.guard -= 1
  │    → return PREVENTED_BY_GUARD
  │
  ③ 军人保护检查（主人公专属）
  │    state.soldier_protection_active
  │    → return PREVENTED_BY_SOLDIER
  │
  └─ ④ 真实死亡
       character.is_alive = False
       → return DIED

DeathResult 枚举：
  DIED | PREVENTED_BY_GUARD | PREVENTED_BY_IMMORTAL | PREVENTED_BY_SOLDIER
```

## 六、PhaseHandler 信号机制

```
PhaseHandler.execute(state) → PhaseSignal

PhaseSignal 三种：
  ├─ PhaseComplete()           正常完成 → Controller 调 StateMachine.advance()
  ├─ WaitForInput(             挂起等待玩家 → Controller 保存 callback
  │    input_type: str,        UI 调 provide_input(choice) 后继续
  │    prompt: str,
  │    options: list,
  │    callback: Callable)
  └─ ForceLoopEnd(             任意阶段强制结束轮回
       cause: str)             → StateMachine.force_loop_end() → 跳 LOOP_END

已实现的 PhaseHandler（phase_base.py）：
  MastermindActionHandler    — 等待剧作家选 3 张行动牌
  ProtagonistActionHandler   — 等待当前主人公选 1 张行动牌
  PlaywrightAbilityHandler   — 剧作家能力阶段（当前骨架）
  ProtagonistAbilityHandler  — 主人公能力阶段（当前骨架）
  IncidentHandler            — 事件判定与原子结算
  LeaderRotateHandler        — 轮换主人公领队（无等待）
  LoopEndHandler             — emit LOOP_ENDED + 保存 LoopSnapshot
  FinalGuessHandler          — 最终推理阶段（等待玩家输入）
```

## 十一、2026-04-19 架构更新记忆

- `AbilityResolver` 已升级为统一入口，按来源分层收集：
  `collect_goodwill_abilities()` / `collect_identity_abilities()` /
  `collect_rule_abilities()` / `collect_derived_abilities()`。
- `collect_character_abilities()` 保留为兼容接口，但语义已收束为“身份能力收集”别名。
- 能力限次统一落在 `GameState.ability_runtime`（`AbilityRuntimeState`）：
  `usages_this_loop` + `usages_this_day`，并在 `advance_day/reset_for_new_loop` 清理。
- `AbilityResolver` 新增 `is_ability_available()` / `mark_ability_used()`：
  能力候选过滤与计次走同一套规则。
- `AbilityResolver` 新增 `resolve_targets()` 基础 selector：
  `same_area_any` / `any_character` / `any_board` / `condition_target` / `hospital_all`。
- EventBus 触发链更新：
  `LoopEndHandler` 负责发布 `LOOP_ENDED`，`AtomicResolver` 在身份 `ON_DEATH` 能力入队前发布 `ABILITY_DECLARED`。

## 八、完整调度流（GameController 主循环）

```
start_game(state)
  │ state_machine.reset() → GAME_PREPARE
  └─ _run_phase()
         │
         ├─ GAME_END        → _handle_game_end() → UICallback.on_game_over()
         ├─ NEXT_LOOP       → state.reset_for_new_loop() → _advance_and_run()
         │
         ├─ 无 handler      → _advance_and_run()（直接推进）
         │
         └─ 有 handler      → signal = handler.execute(state)
                               │
                               ▼ _handle_signal(signal)
                               ├─ PhaseComplete   → _advance_and_run()
                               │                       └─ StateMachine.advance(
                               │                            is_final_day,
                               │                            failure_reached,
                               │                            is_last_loop,
                               │                            protagonist_dead,
                               │                            has_final_guess)
                               │                       → _run_phase()  [递归推进]
                               │
                               ├─ WaitForInput    → _pending_callback = wait.callback
                               │                   → UICallback.on_wait_for_input(wait)
                               │                   [引擎挂起]
                               │                   …玩家操作…
                               │                   provide_input(choice)
                               │                       → callback(choice) → signal
                               │                       → _handle_signal(signal)  [续跑]
                               │
                               └─ ForceLoopEnd    → state_machine.force_loop_end()
                                                  → _advance_and_run()
```

## 九、信息边界（Visibility）

```
Visibility.filter_for_role(state, role) → VisibleGameState

  PlayerRole.MASTERMIND    → 完整信息（identity_id 可见）
  PlayerRole.PROTAGONIST_* → 过滤版本：
      ├─ 未公开角色的 identity_id → "???"
      ├─ 能力触发原因不暴露
      └─ 服务端保留完整 EventBus.log，客户端按此边界过滤

VisibleGameState：
  与 GameState 结构相同，但 CharacterState.identity_id
  对主人公视角未公开角色返回遮蔽值
```

## 十、跨组件数据流总览

```
玩家输入
  │  provide_input(choice)
  ▼
GameController
  │  callback(choice) → PhaseSignal
  │
  ├──► StateMachine        纯流转，输入：条件标志，输出：下一 GamePhase
  │
  ├──► GameState           数据读写（直接字段访问）
  │
  ├──► PhaseHandler        execute(state) → PhaseSignal
  │         └──► AtomicResolver.resolve(state, effects)
  │                   ├─ ① 读：_plan_effect(snapshot) → Mutation[]
  │                   ├─ ② 写：_apply_mutation(state)
  │                   │         └──► DeathResolver.process_death()
  │                   │         └──► EventBus.emit(TOKEN_CHANGED / …)
  │                   └─ ③ 触：_process_triggers() → ResolutionResult
  │                              └──► EventBus.emit(CHARACTER_DEATH / …)
  │
  ├──► EventBus            广播事件 → 日志 / 能力触发（Phase 1 待接入）
  │
  └──► Visibility          过滤 GameState → VisibleGameState → UICallback
```
