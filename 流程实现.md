# 惨剧轮回引擎 — 逐阶段流程实现

> 仅覆盖引擎层，不含 UI

---

## 启动

```python
GameController.__init__()
  # 按依赖顺序创建：
  EventBus → DeathResolver → AtomicResolver(bus, death)
  StateMachine → GameState → Visibility
  create_phase_handlers(bus, atomic) → dict[GamePhase, PhaseHandler]

start_game(state)
  self.state = state
  state_machine.reset()               # current_phase = GAME_PREPARE
  state.current_phase = GAME_PREPARE
  _run_phase()                        # 进入主循环
```

---

## 主循环 `_run_phase()`

每次调用的逻辑：

```
当前 phase = state_machine.current_phase

  GAME_END   → _handle_game_end()，结束
  NEXT_LOOP  → state.reset_for_new_loop()，然后 _advance_and_run()
  无 handler → _advance_and_run()（直接跳过）
  有 handler → signal = handler.execute(state)
               → _handle_signal(signal)
```

`_handle_signal` 三路：

```
PhaseComplete  → _advance_and_run()
                   └─ state_machine.advance(...) → 新 phase
                   └─ _run_phase()               ← 递归

WaitForInput   → _pending_callback = wait.callback
                 挂起，等 provide_input(choice)
                   └─ callback(choice) → signal → _handle_signal()  续跑

ForceLoopEnd   → state_machine.force_loop_end()  # _pending_loop_end = True
                 → _advance_and_run()
                   └─ advance() 检测到 flag → 跳 LOOP_END
```

### LOOP_END
```python
LoopEndHandler.execute(state)
  # 结算 timing=loop_end 的能力 / 失败条件
  state.save_loop_snapshot()   # 保存跨轮快照（角色状态/EX槽/事件记录）
  return PhaseComplete()

# LOOP_END 完成后由 advance() 在本阶段末尾判断：
_branch_after_loop_end(failure_reached, protagonist_dead, is_last_loop, has_final_guess)
  !has_failed                → GAME_END（主人公胜利）
  has_failed + !is_last_loop → NEXT_LOOP
  has_failed + is_last_loop  → FINAL_GUESS / GAME_END
```

---

## 当前可运行路径总结

| 阶段 | 状态 |
|---|---|
| GAME_PREPARE → LOOP_START → TURN_START | 可跑通 |
| MASTERMIND_ACTION | 可等待输入、回调推进 |
| PROTAGONIST_ACTION | 仅队长一人，可等待输入 |
| ACTION_RESOLVE / PLAYWRIGHT_ABILITY / PROTAGONIST_ABILITY / INCIDENT / TURN_END | 空实现，直接 `PhaseComplete` |
| LEADER_ROTATE / LOOP_END / NEXT_LOOP | 可跑通 |
| LOOP_END 末尾分支 | 分支逻辑完整，失败条件由 LOOP_END 能力结算写入 |
| FINAL_GUESS | 接收输入但不判定 |
| GAME_END | 可判定胜负 |
